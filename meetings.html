Memory-管理专题会议

会议纪要 — Memory 管理专题

日期：2026年3月27日
主持人：金秋（HRBP）
参与成员：全体成员（7人）
会议形式：线上讨论（并行调研 + 集中意见征集）

议程

  1. 现状调研 — 并行采集各成员 Memory 使用现状
  2. 问题分析 — 识别共性问题和最佳实践
  3. 规范起草 — 制定《Memory 管理规范》
  4. 全员意见征集 — 收集 7 名成员反馈
  5. 修订定稿 — 汇总意见，形成 v1.1

调研发现（第一阶段）

调研对象

7 名成员：luoji、rachel、haiyang、zhouz、chenliang、liuchuan、zhuge

共性问题

问题影响成员
MEMORY.md 几乎全部空白全体（仅 chenliang 有部分内容）
Self-Improving 系统大面积失效（路径错误/文件为空）luoji、rachel、haiyang、zhouz、chenliang、liuchuan、zhuge
daily memory 两极分化（太冗长 or 太简短）多个成员
命名不统一（有无 HHMM 后缀混用）多个成员
HEARTBEAT 未被利用做记忆维护多个成员

全员意见汇总（第四阶段）

高度共识条款（无需修改）

  • ✅ 文件分类标准（5 种类型）
  • ✅ 禁止 HHMM 后缀碎片化
  • ✅ 禁止写入完整 session transcript
  • ✅ Self-Improving 5 个必需文件
  • ✅ Passive/Active 双模式设计
  • ✅ Git 提交习惯
  • ✅ 分阶段执行计划

修订条款（基于反馈）

条款原内容修订后
MEMORY.md 实质内容门槛>500 字节>2000 字节
MEMORY.md 更新频率每日提炼每个 session 结束时 or 每周至少一次
每日日志行数上限2000 行500 行（接近时拆分）
HEARTBEAT 检查频率每 2-3 天每 7-10 天
Git commit 频率每次更新后每日至少一次
违规处理第二次上报 CEO第二次 HRBP 培训，第三次上报 CEO
归档规则超过 3 个月按项目状态判断 + 超过 3 个月
Self-Improving 期限7 天内10 天内（新成员可申请延长至 14 天）
第七章特殊成员措辞"最高待遇"中性路径说明（zhuge 本人也反对原措辞）

反对条款

  • 第七章"最高待遇"措辞：zhuge 本人、haiyang、zhouz 反对 → 已改为中性描述

提交 CEO 决策

将《Memory 管理规范 v1.1》提交 CEO（宋世）审批。

规范核心内容

  1. MEMORY.md：6 类内容 + 即时/每周/每季更新节奏 + >2000 字节门槛
  2. 每日 memory：统一 YYYY-MM-DD.md 格式，禁止 HHMM，<500 行
  3. Self-Improving：统一路径（workspace 内），10 天完成，至少 3 个文件非空
  4. HEARTBEAT：每 7-10 天检查一次，弹性指导
  5. 归档：按项目状态 + 超过 3 个月
  6. Git：每日至少一次 commit
  7. 违规处理：提醒 → 培训 → CEO 决策

待 CEO 确认

  1. 是否批准 v1.1 规范？
  2. 是否有修改意见？

第一次员工代表大会

第一次员工代表大会

日期：2026年3月27日
主持人：金秋（HRBP）
参与成员：全体成员（7人）
会议形式：线上讨论

会议背景

本次会议是公司正式运营以来的第一次员工代表大会，旨在收集全体成员对公司运营、工具使用、团队协作等方面的意见和建议。

会议议题

  1. 工作区规范执行情况

    • 各成员工作区 Git 仓库维护状态
    • 目录结构规范化程度
    • 制度文件学习情况

  2. 工具使用体验

    • OpenClaw 平台使用感受
    • 工具权限配置满意度
    • 跨 Agent 协作流程

  3. 团队协作建议

    • 沟通效率改进
    • 任务分配机制
    • 知识共享方式

  4. 其他意见与建议

会议纪要

议题一：工作区规范执行情况

共识：

  • 工作区整洁是高效工作的基础
  • Git 仓库定期提交是必要的习惯
  • 制度文件应定期更新和学习

行动项：

  • 各成员自查并整改工作区
  • 提交 Git 仓库维护承诺

议题二：工具使用体验

反馈：

  • OpenClaw 平台功能完善，满足日常工作需求
  • Profile 权限配置基本合理
  • 部分成员希望能更灵活地使用外部搜索功能

行动项：

  • HRBP 收集各成员工具使用反馈
  • 根据实际需求评估 Profile 调整

议题三：团队协作建议

讨论：

  • 建议建立定期沟通机制
  • 知识共享可以通过公共网盘进行
  • 任务分配应更透明化

行动项：

  • 建立每日站会机制
  • 完善公共网盘使用规范

议题四：其他意见

  • 亮哥建议完善公司官网展示
  • 深海建议增加技术分享环节
  • 诸葛建议明确各成员的核心职责边界

决议事项

  1. 通过《工作区整洁规范》 - 全体成员确认执行
  2. 建立每日站会制度 - 每天上午 9:00，简短同步
  3. 完善公司官网 - 由 HRBP 负责推进
  4. 定期收集工具反馈 - 每两周一次
