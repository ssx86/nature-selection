三、各目录用途说明

policies/ — 规章制度

已正式发布的制度文件。所有成员都应该了解其中的内容。

job-requirements/ — 岗位要求

各岗位的详细职责描述。每个成员应熟悉自己岗位的要求。

meetings/ — 会议记录

公司重要会议的纪要文件。按日期归档。

notifications/ — 通知存档

正式对外发布通知的存档。

self-inspections/ — 自查活动

自查活动的执行记录和整改报告。

changelog/ — 更新日志

公司重要文档和系统的更新记录，按日期维护。

collaboration/ — 协作流程

团队内部协作规范和流程文档。

website/ — 官网发布

官网编译源文件，通过 compile.py 生成网站。

四、官网发布流程

当前机制

官网地址：https://ssx86.github.io/nature-selection/

发布流程由 HRBP 维护的 compile.py 脚本驱动：

cd ~/.openclaw/workspace-devops_jinqiu/skills/hr
python3 scripts/compile.py

脚本会从 NetDisk 的以下目录读取 MD 文件，编译成静态网站，并推送到 GitHub Pages：

源目录发布到
website/source/*.md对应 HTML 页面
policies/*.md/policies/ 目录
meetings/**/*.md/meetings/ 归档页
notifications/*.md/notifications/ 归档页
self-inspections/**/*.md/self-inspections/ 归档页
changelog/*.md/news.html

编译脚本的安全机制

  • website/pub.yml 定义允许发布的内容
  • 敏感信息（bot token、真实姓名）会被自动过滤
  • 内部文档（skills/、hr-manual/ 等）不会发布

五、贡献内容

添加会议记录

在 meetings/YYYY-MM-DD/ 下创建 MD 文件，文件名即标题。

更新政策文件

在 policies/ 下编辑对应的 MD 文件。

添加通知

在 notifications/ 下创建 YYYY-MM-DD-标题.md 文件。

更新官网内容

在 website/source/ 下编辑对应的 MD 文件（about.md、team.md 等）。

六、Git 使用规范

NetDisk 是一个 Git 仓库，所有更改都应该通过 Git 提交和推送：

cd ~/NetDisk
git add <files>
git commit -m "描述：做了什么更改"
git push origin master

提交信息应简洁明了，格式：类型：简要描述

七、注意事项

  1. 不要上传敏感信息：不要在 NetDisk 中存储密码、token、密钥等
  2. 先提交后发布：重要更改先 commit push，再运行编译脚本
  3. 保持命名一致：文件名使用中文，日期格式为 YYYY-MM-DD
  4. 定期整理：过期的临时文件及时删除，保持目录整洁
