工作区整洁规范

第一章：目录结构

工作区应包含以下目录结构：

~/.openclaw/workspace-<agentId>/
├── active/              # 开发中/正在使用的文件
│   ├── projects/       # 当前项目文件
│   ├── drafts/          # 草稿、未完成的内容
│   ├── working/         # 正在编辑的文件
│   └── references/      # 参考资料、临时文件
├── archive/             # 过时/归档的文件
│   ├── projects/        # 已完成的项目
│   ├── drafts/          # 未完成但已放弃的草稿
│   └── old/             # 旧版本的参考文件
├── resources/           # 静态资源（图片、模板等）
└── memory/              # 记忆文件

第二章：文件命名规范

2.1 日期格式

使用 YYYY-MM-DD 格式：

  • 2026-03-27-report.md
  • 2026-03-27-notes.md

2.2 版本号

使用 v1.0, v1.1, v2.0 格式：

  • proposal-v1.0.md
  • proposal-v2.0.md

2.3 描述性名称

使用清晰的描述性名称：

  • ✅ 2026-03-27-项目方案.md
  • ✅ meeting-notes-2026-03-27.md
  • ❌ aaa.md
  • ❌ 新建文件.md

第三章：Git 仓库管理

3.1 初始化

每个工作区都应初始化 Git 仓库。

3.2 .gitignore 配置

应忽略以下文件：

  • 隐私文件
  • 技能文件
  • 临时文件
  • 敏感配置

3.3 提交规范

  • 提交频率：每周至少 2 次
  • 提交信息：清晰描述变更内容
  • 重要版本：打标签标记

第四章：定期整理

4.1 每日整理

  • 清理临时文件
  • 整理 working/ 目录

4.2 每周整理

  • 整理 active/ 目录
  • 归档已完成项目

4.3 每月整理

  • 全面检查工作区
  • 清理过期文件
  • 更新归档
