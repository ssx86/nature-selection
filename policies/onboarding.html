新同事入职流程

第一章：入职前准备

1.1 HRBP 工作

当有新同事加入时，HRBP 负责：

  1. 创建 Agent 账号

    • 配置 openclaw.json
    • 设置 Agent ID
    • 配置 Profile

  2. 初始化工作区目录结构

    ~/.openclaw/workspace-<agentId>/
├── active/
│   ├── projects/
│   ├── drafts/
│   ├── working/
│   └── references/
├── archive/
│   ├── projects/
│   ├── drafts/
│   └── old/
├── resources/
└── memory/

  3. 初始化 Git 仓库

    • 创建 .gitignore
    • 初始提交

  4. 配置 Profile

    • 添加工具权限
    • 设置默认模型

1.2 相关同事通知

HRBP 应提前通知相关同事有新成员加入。

第二章：入职第一天

2.1 介绍团队

  • 介绍全体成员
  • 说明各人职责
  • 展示协作模式

2.2 讲解工作流程

  • 日常工作流程
  • 任务分配机制
  • 汇报方式

2.3 工具培训

  • OpenClaw 基本操作
  • 工具权限说明
  • Profile 配置

第三章：试用期管理

试用期（第1-4周）安排：

周次内容目标
第1周基础培训熟悉工作环境和工具
第2周实践培训完成简单任务
第3周进阶培训独立完成任务
第4周评估转正综合评估
